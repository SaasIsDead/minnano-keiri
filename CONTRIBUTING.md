# コントリビューションガイド

「Choreia Books」へのコントリビュートに興味を持っていただきありがとうございます！
このドキュメントでは、開発環境の構築から、プルリクエストの作成までの手順を説明します。

## 開発環境のセットアップ

### 必要なツール

- **Rust**（最新stable）— [rustup.rs](https://rustup.rs/) からインストール
- **wasm-pack** — `cargo install wasm-pack` でインストール
- **ローカルHTTPサーバー** — `python3 -m http.server` や [miniserve](https://github.com/svenstaro/miniserve) など

### 環境構築手順

```bash
# 1. リポジトリをフォーク＆クローン
git clone https://github.com/<あなたのユーザー名>/Books.git
cd Books

# 2. WASMビルド
wasm-pack build --target web

# 3. ビルド出力をwebディレクトリにコピー
cp -r pkg web/pkg

# 4. ローカルサーバーで起動
cd web
python3 -m http.server 8080
```

ブラウザで http://localhost:8080 にアクセスして動作確認してください。

### kaikei-rsをローカルで開発する場合

税金計算ロジック（kaikei-rs）も同時に修正する場合は、`Cargo.toml` でローカルパスを指定できます。

```toml
[dependencies]
kaikei-rs = { path = "../kaikei-rs" }
```

## ビルド方法

```bash
# WASMビルド（開発用）
wasm-pack build --target web

# WASMビルド（リリース用・最適化あり）
wasm-pack build --target web --release

# ビルド出力をwebディレクトリにコピー
cp -r pkg web/pkg
```

`web/` ディレクトリをHTTPサーバーで配信すれば、ブラウザから利用できます。

## コントリビュートの流れ

1. **Issue を確認** — 既存のIssueを確認するか、新しいIssueを作成してください
2. **フォーク** — リポジトリをフォークします
3. **ブランチ作成** — `git checkout -b feature/機能名` のように、わかりやすいブランチ名を付けてください
4. **実装** — コードを変更・追加します
5. **動作確認** — ローカルでビルド・動作確認してください
6. **コミット** — わかりやすいコミットメッセージを書いてください
7. **プッシュ** — `git push origin feature/機能名`
8. **プルリクエスト** — GitHub上でPRを作成してください

## コードスタイル

### Rust

- `cargo fmt` でフォーマットしてからコミットしてください
- `cargo clippy` で警告が出ないようにしてください
- 関数やモジュールには日本語のドキュメントコメント（`///` や `//!`）を付けてください
- パブリックAPIにはテストを書いてください

### HTML / CSS / JavaScript

- インデント: スペース2つ
- 日本語のUIテキストはHTMLに直接記述（i18n対応は将来課題）
- ブラウザ互換性: モダンブラウザ（Chrome, Firefox, Safari, Edge）の最新版をサポート

## Issue の報告

バグ報告や機能提案は [Issues](https://github.com/Choreia/Books/issues) からお願いします。

### バグ報告のテンプレート

- **現象**: 何が起きたか
- **期待される動作**: 本来どうなるべきか
- **再現手順**: どの操作で再現するか
- **環境**: OS、ブラウザ、バージョン

### 機能提案

- **概要**: どんな機能か
- **モチベーション**: なぜ必要か
- **実装案**: 技術的にどう実現するか（わかる範囲で）

## ロードマップ（貢献を歓迎する機能）

以下の機能を実装予定です。興味のある方はぜひPRを送ってください！

- **証憑管理（電子帳簿保存法対応）** — 領収書・請求書のスキャン画像を保存・管理し、電子帳簿保存法の要件を満たす
- **固定資産管理** — 減価償却の自動計算と固定資産台帳の管理
- **見積書・納品書** — 請求書に加え、見積書・納品書のテンプレートと作成機能
- **確定申告書作成支援** — 仕訳データから確定申告書（所得税・消費税）の下書きを自動生成
- **テストの充実** — Rustユニットテスト、WASMインテグレーションテスト、UIのE2Eテスト
